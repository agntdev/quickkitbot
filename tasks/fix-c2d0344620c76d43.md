# fix-c2d0344620c76d43 — T02 spec uses invalid `send` format — `callback_query` is not a recognized SendShorthand variant

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 QKIT

`tests/specs/T02.json` step 2 uses `"send": { "callback_query": { "data": "help" } }`. The `SendShorthand` discriminated union in `src/toolkit/harness/types.ts` only recognizes `{ text, chatId?, userId? }`, `{ callback, chatId?, userId?, messageId? }`, and `{ update }`. There is **no `callback_query` key**. In `runner.ts:86`, `shorthandToUpdate()` checks `"update" in send` (no), then `"text" in send` (no), then falls through to `callbackUpdate(updateId, send.callback, …)` with `send.callback === undefined`. The resulting callback update carries `undefined` data, so `composer.callbackQuery("help", …)` in `start.ts` will never match.

**Fix:** Change the send to `{ "callback": "help" }`.
